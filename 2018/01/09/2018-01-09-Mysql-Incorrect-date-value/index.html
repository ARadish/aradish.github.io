Mysql ERROR 1292 (22007) Incorrect date value 0000-00-00

发表于 2018-01-09 | 分类于 Datebase

自己的用的 MySQL 5.7.18,表中字段为date、datetime类型设置默认值 ‘0000-00-00’、’0000-00-00 00:00:00’ 会出现下面的报错

在 Mysql 手册中看到了这个(传送门)我仿佛发现了解决办法
使用下面两句查看当前全局和会话的 sql model

看到了 “NO_ZERO_IN_DATE” 和 “NO_ZERO_DATE” 这两个模式，具体两个模式的作用如手册所述。
接下来是的事情想都不用想肯定吧这两个模式干掉咯

可以只设置全局的 sql model ,重新连接一次数据库 会话的 sql model 就会和全局的一样了。
去掉了 NO_ZERO_IN_DATE、NO_ZERO_DATE 就可以插入 date、datetime类型设置默认值 ‘0000-00-00’、’0000-00-00 00:00:00’ 了。
